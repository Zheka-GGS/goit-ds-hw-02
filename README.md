# Система управління завданнями

Цей проект реалізує систему управління завданнями з використанням SQLite бази даних. Проект включає створення таблиць, наповнення даними та виконання необхідних запитів.

## Структура бази даних

База даних містить три таблиці:

### Таблиця `users`
- `id`: Первинний ключ, автоінкремент
- `fullname`: Повне ім'я користувача (VARCHAR(100))
- `email`: Електронна адреса користувача, унікальна (VARCHAR(100))

### Таблиця `status`
- `id`: Первинний ключ, автоінкремент
- `name`: Назва статусу, унікальна (VARCHAR(50))

### Таблиця `tasks`
- `id`: Первинний ключ, автоінкремент
- `title`: Назва завдання (VARCHAR(100))
- `description`: Опис завдання (TEXT)
- `status_id`: Зовнішній ключ до таблиці status
- `user_id`: Зовнішній ключ до таблиці users

## Файли проекту

- `create_tables.sql` - SQL-скрипт для створення таблиць бази даних
- `seed.py` - Python-скрипт для наповнення бази даних випадковими даними
- `queries.sql` - SQL-запити для виконання всіх вимог завдання
- `test_queries.py` - Python-скрипт для тестування всіх запитів
- `run_project.py` - Основний скрипт для запуску всіх функцій проекту

## Встановлення та використання

1. Встановіть необхідні залежності:
```bash
pip install faker
```

2. Створіть базу даних та таблиці:
```bash
sqlite3 task_management.db < create_tables.sql
```

3. Наповніть базу даних випадковими даними:
```bash
python seed.py
```

4. Протестуйте всі запити:
```bash
python test_queries.py
```

5. Або запустіть повну демонстрацію проекту:
```bash
python run_project.py
```

## Конфігурація

Шлях до бази даних можна налаштувати за допомогою змінної середовища `DB_PATH`. За замовчуванням використовується файл `task_management.db`.

Приклад встановлення шляху до бази даних:
```bash
export DB_PATH=/path/to/your/database.db
python seed.py
```

## Особливості реалізації

- Унікальність полів `email` у таблиці `users` та `name` у таблиці `status`
- Каскадне видалення завдань при видаленні користувача
- Обмеження зовнішнього ключа для статусу, що запобігає випадковому видаленню статусів
- Використання бібліотеки Faker для генерації реалістичних тестових даних
- Гнучка конфігурація шляху до бази даних через змінні середовища
